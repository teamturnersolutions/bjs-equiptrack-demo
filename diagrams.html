4. Technical Overlay Lifecycle

    graph TD
        KeyEvt[1. Keyboard Event Listener] -->|Detects Ctrl + Space| SetOpen[2. Toggle isOpen state]
        SetOpen -->|isOpen = True| MountOverlay[3. Mount HTML Overlay on Page]
        MountOverlay -->|Triggers useEffect| InitSpeech[4. Initialize Browser Speech Recognition]
        InitSpeech -->|On Microphone Start| SetListeningState[5. Set status = 'Listening...' & isListening = true]
        SetListeningState -->|Speech Ends| Transcribe[6. Transcribe speech to text string]

        BackendAPI -->|Returns Message & Success| ShowResult[9. Display Result Message in Overlay]
        ShowResult -->|Speech Synthesis| SpeakFeedback[10. Speak confirmation aloud via speakers]
        SpeakFeedback -->|Timeout| AutoClose[11. Set isOpen = false to unmount overlay]

6. Backend API Route logic

    graph TD
        Request[1. POST Request to /api/agent] -->|Body: prompt text| CallOllama[2. POST to Local Ollama Chat API]
        CallOllama -->|Ollama JSON format| ParseOllama[3. Extract & JSON.parse Model Output]
        ParseOllama -->|Parsed Object| ValidateKeys[4. Extract: intent, item, person]
        
        ValidateKeys -->|Check intent| IntentBranch{Intent Action?}
        IntentBranch -->|CHECK_IN| FindItemInDB[5. Find Item in DB by Name]
        IntentBranch -->|CHECK_OUT| FindItemInDB
        IntentBranch -->|UNKNOWN / ERROR| ReturnFail[Fail: Respond with error message]
        
        FindItemInDB -->|If item not found| ReturnItemFail[Fail: Item not found message]
        FindItemInDB -->|If item found| CheckActionType{Intent Action?}
        
        CheckActionType -->|CHECK_IN| RunCheckIn[6. Call checkInEquipment Server Action]
        CheckActionType -->|CHECK_OUT| FindMemberInDB[6. Find Member in DB by Name]
        
        FindMemberInDB -->|If member not found| ReturnMemberFail[Fail: Member not found message]
        FindMemberInDB -->|If member found| RunCheckOut[7. Call checkOutEquipment Server Action]
        
        RunCheckIn --> ReturnSuccess[Success: Send updated status message]
        RunCheckOut --> ReturnSuccess

        style Request fill:#1a1a1a,stroke:#cc0000,stroke-width:2px,color:#fff
        style CallOllama fill:#1a1a1a,stroke:#8e44ad,stroke-width:2px,color:#fff
        style IntentBranch fill:#1a1a1a,stroke:#d35400,stroke-width:2px,color:#fff
        style RunCheckIn fill:#1a1a1a,stroke:#27ae60,stroke-width:2px,color:#fff

7. Technical Sequence Flow

    sequenceDiagram
        autonumber
        actor User as Warehouse Operator
        participant UI as Browser UI Overlay
        participant WS as Web Speech API
        participant API as /api/agent (Next.js route)
        participant LocalLLM as Local Ollama API (localhost:11434)
        participant DB as Prisma Database
        participant SA as Server Actions

        User->>UI: Presses Ctrl + Space
        UI->>UI: Triggers Sleek Neon Overlay & Audio Alert
        UI->>WS: Starts Speech Recognition
        User->>WS: Speaks: "Check out RF 01 to James Turner"
        WS->>UI: Returns text transcription
        UI->>API: POST /api/agent { prompt: "Check out RF..." }
        
        rect rgb(20, 20, 25)
            note over API, LocalLLM: Local Inference (Ollama Llama3/Qwen2.5)
            API->>LocalLLM: POST /v1/chat/completions (OpenAI compatible format with JSON output)
            LocalLLM-->>API: Returns JSON { intent: "CHECK_OUT", item: "RF 01", person: "James Turner" }
        end

        rect rgb(25, 20, 20)
            note over API, DB: Database Entity Resolution
            API->>DB: Find item ID where name contains "RF 01"
            DB-->>API: Returns Item ID (e.g., 42)
            API->>DB: Find member ID where name contains "James Turner"
            DB-->>API: Returns Member ID (e.g., 7)
        end

        API->>SA: Invokes checkOutEquipment(memberId: 7, itemIds: [42])
        SA-->>UI: Returns { success: true, message: "Checked out successfully!" }
        UI->>User: Plays success sound, displays status, and speaks confirmation (Web Speech Synthesis)
